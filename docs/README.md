## Contributing to Docs
When making changes to the project:
1.  Update `HISTORY.md` with your changes.
2.  If you make a significant architectural choice, log it in `DECISIONS.md`.
3.  If you hit a wall or find a bug that takes time to fix, log the solution in `DEAD_ENDS.md`.
